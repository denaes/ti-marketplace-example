# Engineering Management

Skills for engineering managers covering epic planning, story writing, Jira workflows, capacity planning, and release management.

## Skills

| Skill | Description |
|-------|-------------|
| [api-contract-spec](api-contract-spec/) | Checklist and format for API contracts (OpenAPI snippet, request/response, errors) |
| [bug-story-writer](bug-story-writer/) | Structure for bug stories with repro steps, root cause, AC for fix, and regression test |
| [capacity-planning](capacity-planning/) | Capacity math, pod mapping, and cross-pod rules for sprint planning |
| [cursor_prompt_builder](cursor_prompt_builder/) | Generate implementation-ready Cursor IDE prompts for Jira stories |
| [definition_of_ready](definition_of_ready/) | PRD interrogator that identifies missing requirements and product debt |
| [dependency-mapper](dependency-mapper/) | Express story/epic dependencies, Jira link types, and ordering |
| [dispatching-parallel-agents](dispatching-parallel-agents/) | Use when facing 2+ independent tasks that can be worked on without shared state |
| [document-release](document-release/) | Post-ship documentation update syncing README, ARCHITECTURE, and CHANGELOG |
| [epic-stakeholder-summary](epic-stakeholder-summary/) | One-pager format for product/leadership on epic scope, story count, risks, and timeline |
| [epic-summary-writer](epic-summary-writer/) | Standard format for epic-summary.md with title, phase, story list, dependencies, and risks |
| [executing-plans](executing-plans/) | Execute written implementation plans in a separate session with review checkpoints |
| [finishing-a-development-branch](finishing-a-development-branch/) | Guide completion of development work with options for merge, PR, or cleanup |
| [jira-epic-folder-matcher](jira-epic-folder-matcher/) | Resolve Jira epic and child stories for a local epic folder |
| [jira-labels-and-metadata](jira-labels-and-metadata/) | Map story tags to Jira labels, fix version, and components |
| [jira_architect](jira_architect/) | Transform a validated PRD into a high-fidelity Jira Epic with standardized stories |
| [plan-eng-review](plan-eng-review/) | Eng manager-mode plan review for architecture, data flow, edge cases, and test coverage |
| [propose-reporting-category-from-sheet](propose-reporting-category-from-sheet/) | Propose Reporting Category values for epics with empty categories in ESC Metrics |
| [qa_test_planner](qa_test_planner/) | Generate comprehensive Playwright test plans for an epic covering all stories |
| [release-milestone-gates](release-milestone-gates/) | Define what "done" means per phase (Alpha/Beta/GA) and required story types |
| [retro](retro/) | Weekly engineering retrospective with commit history analysis and trend tracking |
| [security-story-checklist](security-story-checklist/) | Structured checklist for security stories (auth, RBAC, PII, rate limiting, compliance) |
| [ship](ship/) | Ship workflow: merge base, run tests, review diff, bump version, update changelog, push, create PR |
| [story-estimation](story-estimation/) | Story point scale and justification rules (1/2/3/5/8, when to split) |
| [story-splitting](story-splitting/) | Split oversized stories into scoped parts with clear dependencies |
| [subagent-driven-development](subagent-driven-development/) | Execute implementation plans with independent tasks using subagents |
| [sync-report-writer](sync-report-writer/) | Template and rules for workspace-Jira sync reports |
| [sync-reporting-category-from-sheet](sync-reporting-category-from-sheet/) | Sync the Reporting Category field in Jira from the ESC Metrics spreadsheet |
| [ti-write-hld](ti-write-hld/) | Write a TI High-Level Design aligned with the TI HLD template |
| [workspace-epic-scanner](workspace-epic-scanner/) | Discover all projects and epic folders under workspace/projects/ with Jira alignment |
| [writing-plans](writing-plans/) | Use when you have a spec or requirements for a multi-step task, before touching code |
