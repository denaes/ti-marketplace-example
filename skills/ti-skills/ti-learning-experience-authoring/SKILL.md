# Thought Industries — learning experience & authoring

## Purpose

Reflect Thought Industries’ public message: make it **easy for learners to find what they need** via **dynamic dashboards** and **compelling catalogs**, and support **mastery** through formats from **quick video** to **comprehensive learning paths**, with **authoring and delivery in the platform**.

## When to Use

- Designing information architecture (catalog, paths, dashboards) for a TI site
- Writing requirements for content types, progression, or discovery
- Differentiating “library” browsing vs guided paths

## Public themes to apply

1. **Discovery** — Learners should quickly find the right content; dashboards and catalog UX are first-class in the story.
2. **Format flexibility** — Same platform supports micro-learning and long-form / path-based programs.
3. **Authoring in platform** — Content is authored and delivered without treating the LMS as a passive file dump (wording per TI marketing).

## Instructions

1. **Start from learner jobs-to-be-done.** For each persona, list: “I need to find X,” “I need to complete Y for compliance/certificate,” “I need to skill up for Z.” Map each to catalog sections, paths, or dashboard widgets.
2. **Define path structure.** For comprehensive programs, specify entry criteria, milestones, optional branches, and completion proof; for quick wins, specify standalone assets and where they surface in the catalog.
3. **Pair with personalization skill.** Tagging and distribution (`ti-content-distribution-personalization`) should align with how catalogs and paths are exposed to segments.
4. **Stay marketing-safe.** Do not assert specific authoring UI names, SCORM behavior, or release features unless the user provides internal specs.

## Quality checklist

- [ ] Addresses findability (catalog/dashboard) not only content production
- [ ] Covers both short-form and path-based journeys where relevant
- [ ] Links or defers to monetization skill if purchase or entitlement gates content
